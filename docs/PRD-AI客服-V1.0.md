# AI 客服系统 - 产品需求文档 (PRD)

**版本：** V1.0 (MVP)
**日期：** 2026-03-01
**作者：** 产品经理

---

## 1. 产品概述

**产品名称：** AI 客服系统
**产品定位：** 为中小企业提供低成本、高效率的智能客服解决方案
**目标用户：** 电商、教育、医疗、SaaS 等需要 24/7 客服的企业

---

## 2. 核心功能需求

### 2.1 功能清单

| 序号 | 功能模块 | 功能描述 | 优先级 |
|------|----------|----------|--------|
| 1 | 智能问答 | 基于大模型理解用户问题并智能回答 | P0 |
| 2 | 知识库管理 | 支持上传 Excel/TXT/JSON 格式知识库 | P0 |
| 3 | 多渠道接入 | 微信、飞书、网站客服组件 | P1 |
| 4 | 人机协作 | AI 无法回答时转接人工客服 | P1 |
| 5 | 对话管理 | 查看历史对话记录 | P1 |
| 6 | 数据统计 | 对话量、满意度、常见问题统计 | P2 |

### 2.2 用户角色

| 角色 | 权限 |
|------|------|
| 管理员 | 知识库配置、渠道管理、数据查看 |
| 客服人员 | 查看对话、转接人工 |
| 终端用户 | 发起咨询、智能/人工对话 |

---

## 3. 业务流程

```
终端用户 → 发起咨询 → AI 判断意图 
                         ↓
              ┌─────────┴─────────┐
              ↓                   ↓
         知识库匹配            无法回答
              ↓                   ↓
         返回答案           转接人工客服
              ↓                   ↓
         评价/关闭          人工回复 → 关闭
```

## 5. 接口设计 (后端)

### 5.1 核心 API

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/chat | POST | 发送消息，获取 AI 回复 |
| /api/knowledge | GET/POST | 知识库 CRUD |
| /api/knowledge/sync | POST | 同步知识库到向量库 |
| /api/channel | GET/POST | 渠道配置 |
| /api/conversation | GET | 对话记录列表 |
| /api/conversation/:id | GET | 详细对话内容 |
| /api/stats | GET | 统计数据 |
| /api/transfer | POST | 转人工客服 |

### 5.2 数据模型

**知识库 (Knowledge)**
```
{
  id: string,
  title: string,
  content: string,
  category: string,
  createdAt: timestamp,
  updatedAt: timestamp
}
```

**对话 (Conversation)**
```
{
  id: string,
  userId: string,
  channel: string,
  messages: Message[],
  status: pending/answered/transferred/closed,
  createdAt: timestamp,
  closedAt: timestamp
}
```

## 7. 非功能性需求

| 需求 | 指标 |
|------|------|
| 响应时间 | AI 回复 < 3 秒 |
| 可用性 | 99.9% |
| 并发 | 支持 100+ 同时会话 |
| 安全 | 数据加密传输、敏感信息脱敏 |

## 9. 风险与对策

| 风险 | 对策 |
|------|------|
| 大模型 API 成本超支 | 设置每日调用限额、缓存策略 |
| 知识库匹配不准 | 持续优化 prompt、增加人工审核 |
| 并发性能不足 | 水平扩展、异步处理 |
